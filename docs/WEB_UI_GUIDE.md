# Web UI (Gradio) для VK Publisher

Web UI — это операторская панель поверх FastAPI API.

## Что работает сейчас

- Проверка health (`/health`).
- Просмотр статистики (`/api/v1/stats`).
- Просмотр/редактирование `social_links.json` через API конфигов.

## Что пока заглушки

- Добавление источников контента из UI.
- Ручное создание поста через UI.
- Лента «последние посты» в production-формате.

Эти функции есть в интерфейсе, но ожидают соответствующие backend endpoint'ы.

---

## Запуск

1) Запустите API:

```bash
uvicorn src.main:app --host 0.0.0.0 --port 8000
```

2) В отдельном терминале запустите UI:

```bash
python src/web_ui.py
```

3) Откройте:

- `http://localhost:7860` — Web UI
- `http://localhost:8000/docs` — Swagger API

---

## Запуск на сервере

```bash
nohup python src/web_ui.py > webui.log 2>&1 &
```

Или через systemd unit (по аналогии с основным сервисом API).

---

## Безопасность

Для публичного доступа не открывайте Gradio напрямую в интернет без reverse proxy.

Рекомендуется:
1. Nginx/Caddy перед UI.
2. Базовая аутентификация.
3. Ограничение IP в firewall.
4. HTTPS (Let's Encrypt).

---

## Рекомендация по развитию UI

Если планируется продакшен-операторка, приоритетно реализовать API endpoint'ы для:
- CRUD источников.
- Создания/планирования постов.
- Просмотра последних публикаций со статусами.
