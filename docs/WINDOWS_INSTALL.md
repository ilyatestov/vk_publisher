# 🪟 УСТАНОВКА VK PUBLISHER НА WINDOWS

Эта инструкция поможет вам установить и запустить VK Publisher на операционной системе Windows.

---

## 📋 ТРЕБОВАНИЯ

- Windows 10/11 (64-bit)
- Python 3.10 или выше
- 2 ГБ свободной памяти
- Доступ в интернет

---

## 🚀 ШАГ 1: УСТАНОВКА PYTHON

### 1.1 Скачайте Python
1. Откройте браузер и перейдите на: **https://www.python.org/downloads/**
2. Нажмите кнопку **"Download Python"** (последняя версия)

### 1.2 Установите Python
1. Запустите скачанный файл `.exe`
2. ⚠️ **ВАЖНО:** Поставьте галочку ✅ **"Add Python to PATH"** перед установкой!
3. Нажмите **"Install Now"**
4. Дождитесь окончания установки

### 1.3 Проверьте установку
Откройте командную строку (Win + R → введите `cmd` → Enter) и выполните:

```bash
python --version
```

Должно отобразиться что-то вроде: `Python 3.11.x`

---

## 🚀 ШАГ 2: СКАЧИВАНИЕ ПРОЕКТА

### Способ А: Через Git (рекомендуется)

#### 2.1.1 Установите Git
1. Перейдите на: **https://git-scm.com/download/win**
2. Скачайте и установите Git (оставьте настройки по умолчанию)

#### 2.1.2 Клонируйте репозиторий
Откройте командную строку и выполните:

```bash
cd %USERPROFILE%
git clone https://github.com/ilyatestov/vk_publisher.git
cd vk_publisher
```

### Способ Б: Через ZIP-архив

1. Перейдите на страницу проекта: **https://github.com/ilyatestov/vk_publisher**
2. Нажмите зеленую кнопку **"Code"**
3. Выберите **"Download ZIP"**
4. Распакуйте архив в удобную папку (например, `C:\vk_publisher`)
5. Откройте эту папку в командной строке

---

## 🚀 ШАГ 3: СОЗДАНИЕ ВИРТУАЛЬНОГО ОКРУЖЕНИЯ

Виртуальное окружение нужно для изоляции зависимостей проекта.

### 3.1 Создайте виртуальное окружение
В командной строке выполните:

```bash
python -m venv .venv
```

### 3.2 Активируйте виртуальное окружение
```bash
.venv\Scripts\activate
```

После активации слева в командной строке появится `(.venv)`.

> 💡 **Примечание:** При каждом новом запуске проекта нужно активировать виртуальное окружение!

---

## 🚀 ШАГ 4: УСТАНОВКА ЗАВИСИМОСТЕЙ

### 4.1 Обновите pip
```bash
python -m pip install --upgrade pip
```

### 4.2 Установите зависимости
```bash
pip install -r requirements.txt
```

Дождитесь окончания установки всех пакетов.

---

## 🚀 ШАГ 5: НАСТРОЙКА КОНФИГУРАЦИИ

### 5.1 Скопируйте файл конфигурации
```bash
copy .env.example .env
```

### 5.2 Получите API токены
Следуйте инструкции: **[VK_API_SETUP.md](docs/VK_API_SETUP.md)**

Кратко:
1. Получите токен VK API
2. Узнайте ID вашей группы
3. Создайте Telegram бота
4. Узнайте свой Telegram ID

### 5.3 Отредактируйте файл .env
Откройте файл `.env` любым текстовым редактором (Блокнот, Notepad++, VS Code) и заполните:

```bash
VK__ACCESS_TOKEN=ваш_токен_vk
VK__GROUP_ID=id_вашей_группы
TELEGRAM__TOKEN=токен_telegram_бота
TELEGRAM__MODERATOR_CHAT_ID=ваш_id_telegram
```

Сохраните файл.

---

## 🚀 ШАГ 6: ЗАПУСК ПРИЛОЖЕНИЯ

### 6.1 Запустите приложение
В командной строке выполните:

```bash
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

### 6.2 Проверьте работу
Откройте браузер и перейдите по адресу:
- **Главная страница:** http://localhost:8000
- **Swagger UI (API документация):** http://localhost:8000/docs
- **Health check:** http://localhost:8000/health

---

## 🐳 АЛЬТЕРНАТИВА: ЗАПУСК ЧЕРЕZ DOCKER DESKTOP

Если у вас установлен Docker Desktop для Windows:

### 6.1 Установите Docker Desktop
1. Перейдите на: **https://www.docker.com/products/docker-desktop/**
2. Скачайте и установите Docker Desktop
3. Перезагрузите компьютер после установки

### 6.2 Настройте .env файл
Заполните файл `.env` как описано в шаге 5.

### 6.3 Запустите через Docker Compose
```bash

```

### 6.4 Проверьте статус
```bash

```

```bash

```

## 🔧 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема: "python не является внутренней или внешней командой"
**Решение:** 
- Переустановите Python с галочкой "Add Python to PATH"
- Или добавьте Python в PATH вручную:
  1. Win + R → `sysdm.cpl` → Enter
  2. Вкладка "Дополнительно" → "Переменные среды"
  3. В "Системные переменные" найдите "Path"
  4. Добавьте: `C:\Users\ВАШ_USER\AppData\Local\Programs\Python\Python311\`

### Проблема: "Access Denied" при активации виртуального окружения
**Решение:**
- Запустите командную строку от имени администратора
- Или выполните: `.venv\Scripts\Activate.ps1` в PowerShell

### Проблема: Ошибка при установке зависимостей
**Решение:**
```bash
pip install --upgrade pip setuptools wheel
pip install -r requirements.txt
```

### Проблема: Порт 8000 уже занят
**Решение:**
- Запустите на другом порту:
```bash
uvicorn src.main:app --reload --host 0.0.0.0 --port 8001
```

### Проблема: Docker не запускается
**Решение:**
- Убедитесь что включена виртуализация в BIOS
- Проверьте что Hyper-V включен (для Windows Pro)
- Переустановите Docker Desktop
