# ✅ Чек-лист перед деплоем vk_publisher

Этот чек-лист поможет убедиться, что проект готов к production-развёртыванию.

---

## 🧪 1. Тестирование

### Unit-тесты
- [ ] Все unit-тесты проходят: `pytest -v tests/unit/`
- [ ] Покрытие кода ≥80% для критических модулей
- [ ] Моки для внешних сервисов (VK API, Telegram, Ollama)

### Integration-тесты
- [ ] Тесты с реальной БД (PostgreSQL): `pytest -v tests/integration/`
- [ ] Тесты Redis-кэша
- [ ] Тесты TaskIQ worker'ов

### End-to-End тесты
- [ ] Полный pipeline: fetch → rewrite → publish
- [ ] Тест отложенных публикаций
- [ ] Тест мультипостинга (VK + Telegram)

### Security-тесты
- [ ] `bandit -r src -ll` — нет уязвимостей
- [ ] `semgrep --config auto src` — нет проблем
- [ ] `pip-audit` — зависимости без уязвимостей
- [ ] SSRF protection проверена
- [ ] XSS/SQL injection валидация

### Performance-тесты
- [ ] Нагрузочное тестирование API (locust/ab)
- [ ] Проверка rate limiting
- [ ] Проверка backpressure при пиковой нагрузке

---

## 🔐 2. Безопасность

### Конфигурация
- [ ] `.env` не закоммичен в Git (добавлен в `.gitignore`)
- [ ] `ENCRYPTION_KEY` установлен и ≥32 символов
- [ ] Пароли БД/Redis изменены на уникальные
- [ ] JWT secret ключ сгенерирован криптографически надёжно

### Сеть
- [ ] Reverse proxy настроен (Nginx/Caddy)
- [ ] HTTPS включён (Let's Encrypt)
- [ ] Firewall настроен (UFW/iptables)
- [ ] Доступ к админским интерфейсам ограничен по IP
- [ ] Rate limiting включён

### Данные
- [ ] Токены шифруются в БД (Fernet)
- [ ] Бэкапы БД настроены (ежедневно)
- [ ] Audit log ведётся для критических действий

### Мониторинг безопасности
- [ ] Fail2Ban установлен и настроен
- [ ] Логи аутентификации сохраняются
- [ ] Alerts на подозрительную активность

- [ ] Prometheus настроен и собирает метрики

- [ ] Проверка VK API credentials

### Redis
- [ ] Redis версия ≥7
